    Large language model 大语言模型

    整理:

    处理文本数据¶

    词嵌入（embedding）¶

    将数据转换为向量格式的概念通常被称为嵌入(embedding)。

    深度学习模型无法处理视频、音频和文本等原始数据格式。 因此，我们使用嵌入模型将这些原始数据转换为密集向量表示，这样深度学习架构就可以轻松理解和处理原始数据。

    其中最早和最流行的例子之一是 Word2Vec 方法。 Word2Vec 训练神经网络架构是通过预测给定目标词的上下文或反之来生成词嵌入。 Word2Vec 架构的主要思想是，出现在相似上下文中的词往往具有相似的含义。 因此，当将词嵌入投影到二维空间中以便于可视化时，可以看到相似的术语聚集在一起

    但大型语言模型（LLMs）通常会生成它们自己的嵌入，这些嵌入是输入层的一部分，并在训练期间更新。 将嵌入作为 LLM 训练的一部分进行优化，而不是使用 Word2Vec 的优势在于，嵌入被优化以适应手头的特定任务和数据。

    对于GPT-2和GPT-3，嵌入大小（通常被称为模型隐藏状态的维度）根据具体的模型变种和大小而变化。 这是性能与效率之间的权衡。 最小的GPT-2（1.17亿参数）和GPT-3（1.25亿参数）模型使用768维的嵌入大小来提供具体示例。最大的GPT-3模型（1750亿参数）使用的嵌入大小为12,288维。

    文本分词(序列化)并转为token ID¶

    字节对编码(Byte pair encoding)¶

    字节对编码（Byte Pair Encoding，简称 BPE）是一种常用的分词算法，特别适用于自然语言处理中的子词级别分词。BPE 分词法的核心思想是通过逐步合并频繁出现的字符或字符序列，来构建一个子词词典，从而将文本分割成子词单元。以下是 BPE 分词法的基本步骤：

    初始化：将文本中的每个字符视为一个独立的符号，并统计每个符号的频率。

    合并频繁符号对：找到文本中最频繁出现的符号对（即相邻的两个符号），将其合并为一个新的符号。更新文本和符号频率。

    重复合并：重复步骤 2，直到达到预定的合并次数或符号词典的大小。

    生成词典：最终生成一个包含所有符号（包括初始字符和合并后的新符号）的词典。

    通过这种方式，BPE 分词法能够有效地处理未登录词（即词典中不存在的词），并且能够在一定程度上减少词典的大小，同时保留较高的词汇覆盖率。

    BPE 分词法的优点包括：

    处理未登录词：能够将未登录词分割成已知的子词，从而提高模型的泛化能力。 词典大小可控：可以通过控制合并次数来调整词典的大小。 适用于多语言：能够处理多种语言，特别是那些具有丰富词形变化的语言。 在实际应用中，BPE 分词法常用于训练语言模型，如 GPT 和 BERT，以提高模型的性能和效率

    将token ID 转换为嵌入向量¶

    位置编码¶

    由于LLM的自注意力机制本身也不关注位置，因此需要将额外的位置信息注入到LLM中。

    为了实现这一点，有两种常用的位置编码方式：相对位置编码和绝对位置编码。

    绝对位置编码与序列中的特定位置相关联。对于输入序列中的每个位置，都会添加一个唯一的位置编码到 token 中，来表示其确切位置。例如，第一个 token 将具有特定的位置编码，第二个 token 将具有另一个不同的位置编码，依此类推。

    相对位置编码不是专注于 token 的绝对位置，而是侧重于 token 之间的相对位置或距离。这意味着模型学习的是 “彼此之间有多远” 而不是 “在哪个确切位置”的关系。这样做的好处是，即使模型在训练过程中没有看到这样的长度，它也可以更好地推广到不同长度的序列。

    两种类型的位置编码旨在增强 LLM 理解 token 顺序和关系的能力，确保更准确和更具上下文意识的预测。它们的选择通常取决于具体的应用程序和正在处理的数据的性质。

    OpenAI 的 GPT 模型使用绝对位置编码，在训练过程中进行了优化，而不是像原始 Transformer 模型中的位置编码那样固定或预定义。这个优化过程是模型训练本身的一部分。

    注意力机制¶

    长序列建模的问题¶

    许多任务如语言翻译在transformers出现之前需要依靠编码器-解码器RNN，编码器部分将整个输入文本处理成一个隐藏状态（存储单元），然后，解码器接收这个隐藏状态来生成输出。RNN在解码阶段不能直接访问编码器的早期隐藏状态，可能导致上下文丢失，特别是在依赖关系可能跨越长距离的复杂句子中。

    编码器-解码器RNN的缺陷促使了注意力机制的设计。

    用关注机制捕捉序列数据依赖性¶

    在自注意力中，“自”指的是机制通过关联单个输入序列中的不同位置来计算注意力权重的能力。它评估和学习输入本身各个部分之间的关系和依赖性，例如句子中的单词或图像中的像素。这与传统的注意力机制形成对比，传统机制关注的是两个不同序列的元素之间的关系，例如在序列到序列模型中，注意力可能在输入序列和输出序列之间。

    在自注意力中，上下文向量起着至关重要的作用。它们的目的是通过结合序列中所有其他元素的信息来创建输入序列（如句子）中每个元素的丰富表示。这对于LLMs至关重要，因为LLMs需要理解句子中单词之间的关系和相关性。
