    Kaggle jane street

    概览¶

    在这场由 Jane Street 主办的比赛中，您将利用从生产系统中获得的真实数据建立一个模型，从而了解成功交易的日常挑战。 这项挑战凸显了金融市场建模的困难，包括胖尾分布、非平稳时间序列和市场行为的突然转变。

    Description¶

    在处理现代金融市场的建模问题时，有很多理由让人相信你试图解决的问题是不可能的。即使撇开金融工具的价格理性地反映了所有可用信息这一信念不谈，你也不得不处理时间序列和分布问题，而这些问题的特性是你在其他类型的建模问题中不会遇到的。分布可能是著名的肥尾分布，时间序列可能是非平稳的，而且数据可能通常不满足很多成功的统计方法所依赖的基本假设。此外，金融市场归根结底是人类的活动，涉及大量个人和机构，这些个人和机构随着技术进步和社会变迁而不断变化，并对出现的经济和地缘政治问题做出反应，因此您可以开始了解其中的困难！

    在这项挑战中，我们要求您利用从我们的一些生产系统中获得的真实数据建立一个模型。这些数据非常贴切地反映了我们每天为在现代金融市场中成功交易而必须做的一些事情。我们收集了与我们运行自动交易策略的市场相关的特征和响应变量，并关注是否拥有良好的基础模型。为了既能解决与我们业务相关的挑战性问题，又能尊重我们交易的专有性和高度竞争性，您会注意到我们对数据中的一些特征和响应者进行了匿名和轻度模糊化处理。这些修改并没有改变当前问题的本质，但却使我们能够为您提供一项艰巨的任务，有意义地说明我们在 Jane Street 所做的工作。

    Jane Street 数十年来一直坚持不懈地在交易的各个方面进行创新，并建立机器学习模型来帮助我们做出决策。这些模型帮助我们每天在全球 200 多个交易场所积极交易数千种金融产品。虽然这项挑战只是 Jane Street 员工日常工作中定量问题的一小部分，但我们非常有兴趣了解 Kaggle 社区将如何应对这项挑战，并与您一起探讨您的问题解决方案！

    评估规则¶

    对提交的信息进行评估的评分函数定义为 responder_6 的样本加权零均值 R 平方得分 (R2)。计算公式如下：

    \[\mathrm{R}^2=1-\frac{\sum w_i(y_i-\hat{y_i})^2}{\sum w_iy_i^2},\]

    其中 \(y_i\) 是第 \(i\) 个样本的响应变量，\(\hat{y_i}\) 是预测值，\(w_i\) 是第 \(i\) 个样本的权重。

    提交文件¶

    提交文件 您必须使用所提供的评估 API 提交参赛作品，以确保模型不会在时间上向前推移。 更多详情，请参阅本示例笔记本。

    数据集说明¶

    竞赛数据集由一组包含 79 个特征和 9 个响应变量 的时间序列组成，这些数据虽已匿名，但代表了真实的市场数据。比赛的目标是预测其中一个响应者（即 responder_6）未来最长 6 个月的走势。

    您必须使用所提供的 Python 评估 API 提交比赛，该 API 按时间顺序提供测试集数据。要使用 API，请按照本手册中的示例操作。(请注意，该 API 与我们在以往预测竞赛中使用的传统时序 API 不同）。

    竞赛阶段和数据更新¶

    根据预测任务，竞赛将分两个阶段进行：

    1. 使用历史数据测试集的模型训练阶段。该测试集约有 450 万行。

    (一个partition_id=0的parquet文件有194万行，行组（是总行数的分区，在较大的文件中可能有不止一个行组，行组具有该组中所有行的列数据，Parquet中的信息是按列组织的）为：7)。

    1. 预测阶段，测试集将在提交截止后收集。该测试集的大小应与第一阶段的测试集大致相同。

    为了帮助您撰写稳健的报告，在模型训练阶段的最后几周，我们将扩展公共测试集，使其包含更接近报告提交截止日期的数据。该扩展集上的预测将不计分。

    在预测阶段开始时，未评分的公共测试集将延长至模型训练阶段的最后一天，私人测试集大约每两周更新一次。每次更新时都将重新评分。

    在预测阶段，评估 API 将提供从公共测试集开始到私有测试集结束的测试数据。您必须在每个时间步进行预测，但在此阶段，只有对私有集的预测进行评分。(如果您愿意，可以对未计分的片段进行 0.0 预测）。

    Discussion¶

    导入数据节约时间和资源的小技巧：¶

    这里的训练数据集分为 10 个部分，每个部分有 1 个 parquet 文件。 大多数使用 pandas 的人会倾向于在一个循环中手动提供文件夹路径，并按顺序导入文件。 但在这里不需要这样做，因为 polars 会自动完成这项工作。 只需使用下面的代码，就可以一次性读入所有相关文件：

    import polars as pl
train = \
pl.scan_parquet(
    f"/kaggle/input/jane-street-real-time-market-data-forecasting/train.parquet"
).\
select(
    pl.int_range(pl.len(), dtype=pl.UInt64).alias("id"),
    pl.all(),
)
    Polars 会自动扫描目录路径，一次性读入 10 个文件。 然后，可以使用 partition_id 列根据自己的需要过滤惰性数据框。

    ```py title = "demo.py" import pyarrow as pa from pyarrow import csv import pyarrow.compute as pc import pyarrow.parquet as pq

    parquet_file = pq.ParquetFile("D:\kaggle\train.parquet\partition_id=0\part-0.parquet")

    dir(parquet_file) parquet_file.schema_arrow metadata = parquet_file.metadata print(metadata) print(metadata.serialized_size) type(metadata) print(parquet_file.schema) group = metadata.row_group(0) print(group) dir(group) vendor_col = group.column(0) print(vendor_col) tip_col = group.column(13) print(tip_col)
